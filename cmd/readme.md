# cmd

`cmd` содержит исполняемые точки входа проекта. Это внешний слой Go-приложения: здесь начинается процесс, читается конфигурация, собираются зависимости и запускаются серверы/команды.

## Иерархия

```text
cmd/
  acg/
    main.go      # будущая точка входа основного бинаря
    readme.md
```

## Правило папки

В `cmd` не должно быть бизнес-логики анализа, SQL, Kubernetes-парсинга или HTTP-обработчиков. `main.go` должен быть тонким:

1. загрузить конфигурацию из `internal/config`;
2. создать инфраструктурные клиенты;
3. собрать приложение через `internal/app`;
4. запустить HTTP API;
5. корректно завершиться по сигналу.

## Связь с планом разработки

Основной бинарь обслуживает все этапы из корневого плана:

- `POST /snapshots` запускает сбор snapshot;
- `POST /analysis-runs` запускает анализ;
- `GET ...` отдаёт addressability graph, ACG и evidence.

Если позже появятся отдельные утилиты, их стоит выносить в соседние папки, например `cmd/acg-migrate` или `cmd/acg-debug`, но MVP лучше держать одним бинарём `cmd/acg`.
