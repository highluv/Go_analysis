# cmd/acg

Точка входа основного Go-бинаря анализатора.

Здесь должен лежать только `main.go` и минимальная сборка приложения:

- чтение конфигурации через `internal/config`;
- создание клиентов PostgreSQL, MinIO и Kubernetes;
- сборка зависимостей из `internal/app`;
- запуск HTTP-сервера;
- обработка сигналов завершения.

Бизнес-логика, SQL-запросы, Kubernetes-парсинг и алгоритмы графа сюда не попадают. `cmd/acg` должен оставаться тонким слоем, который связывает готовые внутренние пакеты в исполняемую программу.

Ожидаемый запуск на этапе разработки:

```bash
go run ./cmd/acg
```

Ожидаемая сборка:

```bash
go build -o bin/acg ./cmd/acg
```
